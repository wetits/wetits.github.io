Quels sont les langages du web?

Le HTML

HTML (HyperText Markup Langage) est un langage de structuration de page en utilisant des balises, la première version est apparue au début des années 1990 avec la version HTML 1.0. Cette première version était très basique et permettait l'utilisation d'hypertexte permettant de naviguer entre différentes pages. En 1995 la version 2.0 de HTML est sortie elle comporte plus de balises que la précédente, notamment des balises d'insertion de tableaux, d'images ou encore de formulaire permettant une interaction avec l'utilisateur. La version 3.0 était axée sur l'amélioration des rendus visuels mais posait des problèmes de compatibilités entre les différents navigateurs. La version 3 qui eut le plus de succès est la 3.2 (WILBUR) en 1996 qui implémente des applets java et javascript permettant l'utilisation de code dans une page web. Vient ensuite la version 4.0 (COUGAR) en 1998 elle améliore des balises comme script ou frame. La version actuelle est la version HTML5 finalisée en 2014, elle implémente de nouvelles balises, et permet de spécifier plus d'APIs en JavaScript.

Le JavaScript

Javascript apparaît en 1995 c'est enfaite le nouveau nom de LiveScript pour Netscape navigator 2.0. JS est un langage de programmation interprété, destiné à l'interaction avec l'utilisateur et la gestion de commandes sur les pages web. Ce langage est apparue dans les navigateur pour gérer les interactions avec les utilisateurs.

Le CSS

CSS (Cascading Style Sheet) est un langage de feuille de style qui permet de faire de la mise en page. Couplé au langage HTML il permet de mettre en page des pages web et de les personnaliser à volonté. Le CSS est apparu en 1990.

Quels sont les outils à dispositions?

Avec la popularisation des sites web et de l’utilisation de HTML/CSS/JS sont arrivés les frameworks. Ce sont des outils de développement qui permettent de standardiser et d’accélérer les développements web en offrant tout un tas de librairies, de structuration de code, de système de sécurités… Ces frameworks se distinguent en deux grandes catégories, les frameworks frontend qui vont être axés sur le développement de code destiné à gérer les interactions clients et les frameworks backend qui vont quand à eux êtres axés sur le développement de serveurs et notamment d’APIs.

Frameworks frontend

Parmi les frameworks frontend, on retrouve entre autre React développé par Meta, Angulaire développé par Google et flask.

Frameworks backend

Parmi les frameworks backend, on retrouve entre autre Django, express ou encore Ruby on rails pour citer les plus connus.

Voici quelques exemples d’application:

Airbnb: react/RoR
Instagram: react native/django
Google: angular

Qu'en est-il des mobiles?

Avec l’évolution des technologies, le portable prend une place de plus en plus grande dans la vie des utilisateurs surtout lorsque ceux-ci cherchent à se connecter à internet. De part sa petite taille et sa simplicité d’utilisation, c’est devenu un moyen privilégié pour rester connecté. Mais malgré son utilité et sa simplicité d’utilisation, réussir à coder des sites ou applications web pour mobile est une toute autre paire de manches. En effet, la taille des écrans étant très différentes des ordinateurs et le fonctionnement étant tout autre, réussir à adapter les techniques classiques n’est pas forcément évident.

Evolution du web mobile

Le web mobile est apparu en 1999 avec le protocole WAP (Wireless Application Protocol). C’est un protocole qui se base sur le langage HTML pour adapter les formats de sites web classiques aux formats des petits écrans de téléphone. Ensuite est apparu en 2005 la technologie haut débit Wi-FI (Wireless Fidelity) pour l’accès à internet qui permet enfin un accès rapide à internet. Puis sont apparu les technologies EDGE, UMTS (3G) et LTE (4G) qui permettent un accès presque partout à internet.

Frameworks hybrides

Les frameworks classiques sont prévus pour un usage classique sur ordinateur mais il existe aussi des frameworks hybrides qui permettent de développer un site web qui sera tout aussi accessible que ce soit sur ordinateur ou bien sur mobile. Les applications développées à partir de ces frameworks sont appelées des applications hybrides qui possèdent une visualisation tout aussi bien pour ordinateur que pour mobile. Pour citer les frameworks hybrides les plus connus on pourrait parler d’Ionic, de React native ou encore de Xamarin.

Application mobile first

Les applications mobiles first sont des applications développées dans un premier temps pour les mobiles. Puis une fois l’application pour mobile bien développée, une version pour ordinateur est développé mais sans vraiment que l’entreprise y accorde beaucoup de temps. En effet le but principal de ces applications est d’être utilisée sur mobile et non sur ordinateur. Mais l’entreprise possédant déjà beaucoup de code, il lui est alors facile et rapide de modifier un peu le code pour créer une version bureau en plus.

Pour citer quelques exemples, on a sworkit: en Ionic et Instagram: en React native.

Il existe aussi des applications mobile-only, ce sont des applications qui ont été développées uniquement pour utilisation sur mobile.

Le no-code qu'est-ce et pourquoi l'utiliser?

Le no-code est une nouvelle manière de programmé, ça se présente sous la forme d’un outils de développement qui permet sans coder purement de développer des sites et applications web.

Les avantages du no-code sont:

- rapide
- peu de connaissance
- faibles coûts de développement

Les inconvénients du no-code sont:

- ces outils ne possèdent pas toutes les fonctionnalités.
- le développeur n’a pas la propriété du code source.
- le développeur manque de contrôle sur l’application finale.

Utilisation

Le no-code de par ses inconvénients majeurs et ses avantages est très utilisé en début de projet pour obtenir rapidement des PoC (Proof of concept) afin de valider avec des potentiels clients le modèle du site ou de l’application web à créer. Ces outils sont aussi beaucoup utilisés dans des sites web « classiques » comme le prototypes de sites type SaaS ou encore la réalisation de CRM (customer relationship management).
